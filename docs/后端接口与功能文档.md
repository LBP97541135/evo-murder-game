# EvoMap Murder Game 后端接口与功能文档

本项目是一个基于 **FastAPI** 构建的“多 Agent 自进化剧本杀系统”，核心逻辑围绕 **EvoMap GEP-A2A** 协议展开，实现了 Agent 的角色扮演、推理互动、记忆进化以及完善的证物与阶段管理。

---

## **后端接口表 (API Reference)**

后端服务统一运行在 FastAPI 框架下，路由按功能模块划分为以下几大类：

### **1. Agent 管理与进化 (`/agents`)**
| 接口地址 | 方法 | 功能描述 | 关键参数 |
| :--- | :--- | :--- | :--- |
| `/agents/register` | POST | 注册新 Agent（同步到 EvoMap 网络或本地降级） | `name`, `role`, `model`, `constitution` |
| `/agents/list` | GET | 获取所有已注册的 Agent 列表 | - |
| `/agents/heartbeat/{key}` | POST | Agent 心跳保活 | `agent_key` |
| `/agents/evolve/{key}` | POST | 进化操作：手动更新 Agent 的宪章或身份文档 | `update_type`, `new_content` |
| `/agents/personas` | GET | 获取预设人设库（按角色筛选） | `role` |
| `/agents/personas/load` | POST | 将指定人设（如“白鸦”）加载并融合到 Agent | `agent_key`, `persona_key` |
| `/agents/personas/auto-match` | POST | 根据剧本类型自动为所有 Agent 匹配人设 | `script_genre`, `difficulty` |

### **2. 游戏流程控制 (`/game`)**
| 接口地址 | 方法 | 功能描述 | 关键参数 |
| :--- | :--- | :--- | :--- |
| `/game/create-session` | POST | 创建游戏会话，初始化引擎、Agent 状态及胶囊注入 | `topic`, `script_id` |
| `/game/session-info/{id}` | GET | **一站式接口**：获取当前阶段、Agent 映射、证物统计等 | `session_id` |
| `/game/phase/{id}/advance` | POST | 推进到下一游戏阶段（触发记忆压缩） | `session_id` |
| `/game/chat/{id}` | POST | 玩家发言：自动封装上下文调用 LLM并分发回复 | `content`, `target_key` |
| `/game/speak-round/{id}` | GET | 获取发言轮次状态（谁在说话、插队队列） | `session_id` |
| `/game/vote/{id}` | POST | 提交投票（指认凶手与动机） | `killer`, `motive` |
| `/game/reveal/{id}` | POST | 游戏结束：生成凶手交代和 DM 真相揭晓 | `killer`, `correct` |
| `/game/snapshot/{id}` | GET | 状态快照：用于前端页面刷新后重建 UI | `session_id` |

### **3. 证物系统 (`/evidence`)**
| 接口地址 | 方法 | 功能描述 | 关键参数 |
| :--- | :--- | :--- | :--- |
| `/evidence/script/{s_id}/session/{g_id}` | GET | 获取本局游戏所有已发现/未发现的证物 | `script_id`, `session_id` |
| `/evidence/present` | POST | 向 Agent 出示证物（触发预设反应或 LLM 动态反应） | `evidenceId`, `presentedTo` |
| `/evidence/combine` | POST | 组合两个证物以生成新线索 | `primaryId`, `secondaryId` |

### **4. 记忆与经验 (`/memory`)**
| 接口地址 | 方法 | 功能描述 | 关键参数 |
| :--- | :--- | :--- | :--- |
| `/memory/record` | POST | 记录 Agent 经验到私有记忆库（胶囊化） | `signals`, `summary`, `score` |
| `/memory/recall` | POST | 召回 Agent 历史经验（用于跨剧本进化） | `signals`, `limit` |

### **5. 用户与助手 (`/users`)**
| 接口地址 | 方法 | 功能描述 | 关键参数 |
| :--- | :--- | :--- | :--- |
| `/users/profile` | GET | 获取用户画像、等级、偏好标签 | `user_id` |
| `/users/assistant/chat` | POST | 个人助手对话：基于用户画像推荐剧本或解答规则 | `message` |

## **后端核心功能实现表**

| 功能模块 | 实现机制与技术要点 | 关键文件参考 |
| :--- | :--- | :--- |
| **三层 LLM 管道** | 采用 `initial → critique → refine` 流程，确保 Agent 回复符合角色性格且不剧透。 | `api/llm/llm_service.py` |
| **自进化机制** | 局后通过 LLM 自动生成反思摘要，并改写 Agent 的 `constitution`（行为宪章），实现长期能力增长。 | `api/agents/agent_orchestrator.py` |
| **游戏引擎 (FSM)** | 维护 `intro` -> `investigation` -> `voting` -> `reveal` 的状态机，控制消息广播与权限。 | `api/agents/game_engine.py` |
| **动态上下文注入** | 根据当前游戏阶段、Agent 记忆、已发现证物动态拼接 Prompt，使 Agent 具备实时感知能力。 | `api/agents/game_context.py` |
| **分级提示系统** | DM-Agent 提供 L1-L4 四级提示（从“提醒目标”到“强暗示”），引导玩家破解僵局。 | `api/agents/agent_orchestrator.py` |
| **证物反应逻辑** | 优先匹配剧本预设反应；若无预设，则由 LLM 模拟角色在看到证物时的惊讶、隐瞒或狡辩行为。 | `api/routes/evidence.py` |
| **发言轮次控制** | 实现复杂的发言顺序管理，支持 Agent 主动插队（Interject）和强制回答（Force Answer）。 | `api/agents/game_engine.py` |

## **开发与调试建议**
- **快速调试**：可以直接通过 `/docs` 查看 FastAPI 自动生成的 Swagger UI，实时测试接口。
- **数据流向**：前端主要通过 `session-info` 和 `snapshot` 接口获取全局状态，通过 `invoke` 或 `chat` 接口驱动对话。
- **扩展开发**：若要增加新功能（如新的游戏机制），建议在 `api/agents/game_engine.py` 中扩展状态逻辑，并在 `api/routes/game.py` 中暴露接口。
