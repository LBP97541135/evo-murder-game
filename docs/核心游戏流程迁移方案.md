## 二、游戏流程重构（EvoMap化）

原项目的游戏流程是 **单Agent本地闭环**，没有跨Agent协作和进化。
evo-murder-game 需要 **多Agent + EvoMap** 闭环。核心变化如下：

### 原流程 → 新流程对照

```
原流程（单Agent本地）                    新流程（多Agent + EvoMap）
─────────────────────────────────    ──────────────────────────────────────
用户提交需求                              用户提交需求
    │                                        │
    ▼                                        ▼
本地LLM直接调用                         DM-Agent 创建 Session
    │                                    邀请所有 Companion-Agent
    │                                        │
    ▼                                        ▼
角色自己用LLM生成回复                    Companion-Agent:
    │                                    1. Memory recall → 召回历史经验
    │                                    2. fetch社区Gene → 获取更好策略
    │                                    3. 三层管道生成回复
    │                                    4. Memory record → 记录本次经验
    │                                        │
    ▼                                        ▼
三层管道:                                 三层管道:（不变）
  initial → critique → refine              initial → critique → refine
    │                                        │
    ▼                                        ▼
直接展示给用户                            Session message 发送给用户
    │                                    其他Agent也可见（广播）
    │                                        │
    ▼                                        ▼
用户推理 → 选凶手 → 选动机              用户推理 → 选凶手 → 选动机（不变）
    │                                        │
    ▼                                        ▼
后游戏自白                               DM-Agent 发起 Council 提案
    │                                    → 五阶段审议 → 决议
    │                                        │
    ▼                                        ▼
结束                                      所有 Agent Memory record 自评
                                          → Companion 改写 constitution（进化）
                                          → 发布 Gene+Capsule 到社区
                                              │
                                              ▼
                                          结束（但 Agent 已进化，下一局更强）
```

### 新增环节（EvoMap独有）

| 环节 | EvoMap API | 触发时机 | 说明 |
|------|-----------|---------|------|
| **游戏开局** | `session/create` | 用户点击开始游戏 | DM创建Session，邀请所有Companion |
| **经验预加载** | `memory/recall` | Companion收到任务时 | 召回"上次类似剧本怎么演"的经验 |
| **社区策略获取** | `fetch`（search_only） | Companion准备生成回复前 | 搜索社区有无更好推理策略 |
| **局内消息** | `session/message` | Companion回复时 | 通过Session发给用户（广播或定向） |
| **证据出示通知** | `session/message`（定向） | 用户出示证据时 | DM监听证据出示，检查是否违规 |
| **局后自评** | `memory/record` | 游戏结束时 | 每个Agent记录本次经验 |
| **constitution改写** | 本地 `update_constitution` | 自评后 | "上次抢话了，这局要少说" |
| **经验发布** | `publish` Gene+Capsule | constitution改写后 | 发布进化经验到社区供其他Agent继承 |
| **决策审议** | `council/propose` + `dialog` | 推理提交后 | DM发起Council：是否通过此推理结果 |

## 三、逐步迁移计划

### 第一阶段：复制核心游戏流程（不改EvoMap逻辑，先让游戏能跑）

**目标**：把 ai-murder-mystery 的核心游戏流程搬过来，先跑起来再说EvoMap。

**从哪搬**：`ai-murder-mystery/web/src/pages/PlayScript.tsx` + `mysteryContext.tsx`

**搬到哪里**：`evo-murder-game/web/src/pages/PlayScript.tsx` + `providers/contexts.tsx`

**改动**：
- `PlayScript.tsx`：直接复制，改import路径
- `mysteryContext.tsx` → `contexts.tsx` 的 MysteryProvider：合并，保留 `actors` 和 `globalStory`

**改动**：

#### Step 3 — 角色对话组件 Actor.tsx（最核心）

### 第二阶段：接入EvoMap（在能跑的基础上加进化）

#### Step 8 — Agent注册 + Session创建

在 `GamePage.tsx` 的"开始游戏"流程中加入：
1. 检查是否有已注册的 DM-Agent 和 Companion-Agent（调用 `/agents/list`）
2. 如果没有，提示用户先去 `/agents` 页面注册
3. 调用 `/game/create-session` 创建EvoMap Session
4. Session ID 传入 `sessionContext`

#### Step 9 — Companion对话时Memory recall

在 `Actor.tsx` 的 `sendChatStream` 中加入：
1. 对话开始前，调用 `recallMemory(nodeId, ["role-playing", scriptType])`
2. 将召回的历史经验注入system prompt（"上次你演类似角色时，用户喜欢简洁回答"）
3. 对话结束后，调用 `recordMemory(nodeId, signals, score, summary)` 记录本次经验

#### Step 10 — 局后自评 + constitution改写

在 `GamePage.tsx` 的 `handleBackToGame` 后游戏流程中加入：
1. 调用 `/game/reflect/{sessionId}` 让所有Agent执行自评
2. 前端展示自评结果（"小七：这局推理评分0.82，比上局提升了0.15"）
3. 可选：让Agent自动改写constitution（调用 `/agents/evolve/{agentKey}`）
4. 前端展示constitution变更（旧→新对比）

#### Step 11 — Council审议

在推理提交后加入：
1. DM-Agent 调用 `council/propose` 发起审议："推理结果是否通过"
2. Companion-Agent 参与 `dialog`（diverge/challenge/vote）
3. 前端展示审议过程（时间线视图）

## 五、风险与依赖

| 风险 | 影响 | 缓解方案 |
|------|------|---------|
| ai-murder-mystery用的是CRA（react-scripts），evo-murder-game也用CRA | 无冲突 | 保持一致的构建工具 |
| Mantine UI v7 API可能与原项目v6有差异 | 组件样式可能需要调整 | 逐组件检查，用Mantine v7等效API替换 |
| 流式SSE在EvoMap中转站是否支持？ | 如果不支持，流式对话无法工作 | 降级为非流式调用（已支持） |
| 三层管道消耗额外token（critique+refine各一次调用） | 积分可能不够 | 添加 `skip_critique` 选项，黑客松Demo可跳过 |
| Companion-Agent注册到EvoMap后，Session消息是异步的 | 用户可能看不到实时流式回复 | 第一阶段先不接EvoMap Session，用本地调用；第二阶段再切换 |
